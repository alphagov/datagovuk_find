# General guidance
Where to start when you’re creating a new public service.

#### [Service Manual](https://www.gov.uk/service-manual)
Any team that’s creating and running a public service should start with
the government’s Service Manual. It explains how to make your service
accessible, work in a flexible way, design the right service, choose
technology and measure success. This will help you meet the Service
Standard – 14 principles for building a good service – and pass a service
assessment, if you need one.

#### [Technology Code of Practice (TCoP)](https://www.gov.uk/guidance/the-technology-code-of-practice)
The TCoP is a set of criteria to help government design, build and buy
technology. You should consider its 13 points for any technology project
or programme as part of the Cabinet Office spend control process.

#### [Make better use of your data](https://www.gov.uk/guidance/make-better-use-of-data)
Linked with the TCoP, this guidance explains how to use data more
effectively by improving your technology, infrastructure and processes. It
explains how making better use of your data will help your programme or
service. It lists the required data regulations and suggests how to
manage your data throughout its life cycle.

#### [Government Analysis Function](https://analysisfunction.civilservice.gov.uk/)
A cross-government network that aims to improve the analytical
capability of the Civil Service. Anyone in government can use the
function to make informed decisions so that policy and operations can
be evidenced and deliver value for money. The function includes guidance, research, training courses and ways to connect with other
analysts.
